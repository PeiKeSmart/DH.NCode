# 统计报表实战（二）：同比环比与异常告警

本文基于 `XCode.Statistics`（`StatModel/StatHelper/StatLevels`）扩展“统计报表二期”能力：

- 同比（YoY）
- 环比（MoM/DoD）
- 异常告警（阈值 + 波动）

> 说明：本文不依赖 `Web/**` 与已在 csproj 排除的统计实体文件，可直接用于你自己的业务统计实体。

---

## 1. 指标口径约定

为了避免前后端口径不一致，先定义统一公式。

### 1.1 同比/环比

设当前值为 $C$，对比基准值为 $B$：

$$
Rate = \frac{C - B}{\max(B, \epsilon)}
$$

其中 $\epsilon$ 取一个很小正数（如 $10^{-6}$），避免分母为 0。

常见转换：
- 同比：`B = 去年同周期值`
- 月环比：`B = 上月同口径值`
- 日环比：`B = 昨日同口径值`

### 1.2 异常告警评分（建议）

把绝对阈值和波动阈值结合：

$$
Score = w_1 \cdot Z + w_2 \cdot \left|Rate\right|
$$

- $Z$：标准化偏离（如与近 $N$ 期均值方差比较）
- `|Rate|`：同比/环比绝对变动率
- `w1/w2`：权重（示例 `0.7/0.3`）

当 `Score > AlertThreshold` 时触发告警。

## 2. 统计模型扩展建议

你的模型可继承 `StatModel<T>` 并增加业务维度：

```csharp
public class BizStatModel : StatModel<BizStatModel>
{
    public String? Biz { get; set; }
    public String? Channel { get; set; }
}
```

统计实体实现 `IStat` 并保留常见指标字段：
- `Times`（量）
- `Users`（人数）
- `Error`（异常数）
- `Cost`（平均耗时）
- `MaxCost`（峰值耗时）

唯一索引建议：`(Level, Time, Biz, Channel)`。

## 3. 写入链路（沿用 GetOrAdd）

在线写入仍使用 `StatHelper.GetOrAdd`，保持并发安全：

```csharp
var model = new BizStatModel
{
    Time = DateTime.Now,
    Biz = "Order",
    Channel = "App",
};

foreach (var m in model.Split(StatLevels.Day, StatLevels.Month, StatLevels.Year))
{
    var st = StatHelper.GetOrAdd<BizStat, BizStatModel>(
        m,
        (x, useCache) => BizStat.FindByKeys((Int32)x.Level, x.Time, x.Biz, x.Channel, useCache),
        e =>
        {
            e.Biz = m.Biz;
            e.Channel = m.Channel;
        });

    if (st == null) continue;

    st.Times++;
    st.Users += 1;
    st.SaveAsync(2000);
}
```

## 4. 同比/环比查询模板

## 4.1 查询当前周期与对照周期

```csharp
var now = DateTime.Today;
var start = now.AddDays(-30);
var end = now;

// 当前区间（日粒度）
var current = BizStat.Search(new BizStatModel { Level = StatLevels.Day, Biz = "Order" }, start, end, null)
    .ToDictionary(e => e.Time, e => e.Times);

// 去年同期（同比）
var yoyBase = BizStat.Search(new BizStatModel { Level = StatLevels.Day, Biz = "Order" }, start.AddYears(-1), end.AddYears(-1), null)
    .ToDictionary(e => e.Time.AddYears(1), e => e.Times);

// 前一周期（环比，这里示例日环比）
var momBase = BizStat.Search(new BizStatModel { Level = StatLevels.Day, Biz = "Order" }, start.AddDays(-1), end.AddDays(-1), null)
    .ToDictionary(e => e.Time.AddDays(1), e => e.Times);
```

## 4.2 计算结果 DTO

```csharp
public sealed class TrendPoint
{
    public DateTime Time { get; set; }
    public Double Current { get; set; }
    public Double YoYRate { get; set; }
    public Double MoMRate { get; set; }
}

static Double Rate(Double current, Double baseline)
{
    var den = Math.Abs(baseline) < 1e-6 ? 1e-6 : baseline;
    return (current - baseline) / den;
}

var points = new List<TrendPoint>();
foreach (var kv in current.OrderBy(e => e.Key))
{
    var t = kv.Key;
    var c = kv.Value;

    yoyBase.TryGetValue(t, out var y);
    momBase.TryGetValue(t, out var m);

    points.Add(new TrendPoint
    {
        Time = t,
        Current = c,
        YoYRate = Rate(c, y),
        MoMRate = Rate(c, m),
    });
}
```

## 5. 异常告警实现（示例）

## 5.1 规则

- 规则 A：当前值超过硬阈值（如 `Times > 100000`）
- 规则 B：环比涨幅超过阈值（如 `MoMRate > 0.5`）
- 规则 C：误差比率异常（如 `Error/Times > 0.03`）

满足任一规则即触发。

## 5.2 代码模板

```csharp
public static IEnumerable<String> CheckAlerts(IEnumerable<TrendPoint> points, IDictionary<DateTime, Double> errors)
{
    foreach (var p in points)
    {
        errors.TryGetValue(p.Time, out var err);
        var errRate = p.Current <= 0 ? 0 : err / p.Current;

        var alerts = new List<String>();
        if (p.Current > 100000) alerts.Add("流量超阈值");
        if (p.MoMRate > 0.5) alerts.Add("环比异常上涨");
        if (p.MoMRate < -0.4) alerts.Add("环比异常下跌");
        if (errRate > 0.03) alerts.Add("错误率异常");

        if (alerts.Count > 0)
            yield return $"{p.Time:yyyy-MM-dd} | {String.Join("、", alerts)}";
    }
}
```

## 6. 报表输出建议

前端展示建议至少包含：

1. 趋势图：当前值、同比%、环比%
2. 异常标记：点位高亮（红/黄）
3. 告警列表：时间、指标、原因、建议动作

后端 API 建议一次返回：
- `points[]`（趋势点）
- `alerts[]`（告警）
- `summary`（区间平均、峰值、异常次数）

---

## 7. 常见坑位

- **时间未规整**：请确保 `StatModel.GetDate(level)` 一致使用，否则同比/环比会错位。
- **唯一索引不完整**：少了维度字段会把不同业务数据叠加。
- **把同比环比放前端算**：多端实现容易漂移，建议后端统一计算。
- **仅看绝对值不看变动率**：容易漏掉“基数小但异常剧烈”的问题。

---

## 8. 生产落地最小闭环

1. 在线埋点写入（`GetOrAdd + SaveAsync`）
2. 每 5 分钟聚合一次趋势快照（可缓存）
3. 每 5 分钟计算告警并推送（邮件/IM）
4. 次日离线复核（校验误报漏报）

这样可在不引入额外中间件的前提下，快速把 XCode 统计能力升级到“可运营”的报表体系。
