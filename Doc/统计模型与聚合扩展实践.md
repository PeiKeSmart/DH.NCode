# 统计模型与聚合扩展实践（Statistics）

`XCode.Statistics` 提供了“按时间层级聚合”的轻量统计模型，适合做访问统计、业务指标汇总、分时看板等场景。

## 核心组件

- `StatModel`
  - `Time`：统计时间
  - `Level`：统计粒度（All/Year/Month/Day/Hour/Minute）
  - `GetDate(level)`：将时间规整到粒度起点
  - `Fill(ps)`：从请求参数反填模型
- `StatModel<T>`
  - `Clone()`：克隆当前模型
  - `Split(levels)`：按多个层级拆分统计任务
- `StatField`
  - 定义聚合字段与模式（`Sum/Count/Avg/Min/Max`）
- `StatHelper.GetOrAdd`
  - 并发安全地“查找或创建”统计实体

## 1）时间层级标准化

统计数据通常要避免同一层级重复计数，`GetDate` 会把时间归一化：

- Year → `yyyy-01-01`
- Month → `yyyy-MM-01`
- Day → `yyyy-MM-dd`
- Hour → `yyyy-MM-dd HH:00:00`
- Minute → `yyyy-MM-dd HH:mm:00`

这保证同一层级键值稳定，可直接作为唯一索引的一部分。

## 2）多层级拆分统计

同一条事件，常需同时写入“日/月/年”多个汇总。

```csharp
var model = new VisitStatModel { Time = DateTime.Now, Level = StatLevels.Day };
var list = model.Split(StatLevels.Day, StatLevels.Month, StatLevels.Year);

foreach (var item in list)
{
    // 分别写入不同层级统计
}
```

## 3）并发下的 GetOrAdd 模式

`StatHelper.GetOrAdd<TEntity,TModel>` 做了两件事：

1. 先查（允许命中缓存）
2. 未命中则插入；若并发插入冲突，二次查询兜底

并自动：
- 设置 `CreateTime`
- 更新 `UpdateTime`
- 标记 AdditionalFields 为脏数据（便于累加字段落库）

这特别适合高并发统计计数。

## 4）典型实体设计建议

统计实体建议字段：

- `Level`（Int32）
- `Time`（DateTime，已规整）
- 若干指标字段（PV/UV/Cost/Sum...）
- `CreateTime` / `UpdateTime`

并建立唯一索引：

- `(Level, Time, 维度键...)`

例如页面访问统计可按 `(Level, Time, Page)` 唯一。

## 5）与 FieldExtension 的配合

在统计查询端，可用 `FieldExtension` 做 group/aggregate：

- `GroupBy()`
- `Count("group_count")`
- `Sum/Avg/Min/Max`

适合构造排行榜、趋势线、TopN 维度分析。

## 6）实践模板

```csharp
var now = DateTime.Now;
var models = new VisitStatModel { Time = now }.Split(StatLevels.Day, StatLevels.Month);

foreach (var m in models)
{
    var entity = StatHelper.GetOrAdd<VisitStat, VisitStatModel>(
        m,
        (x, useCache) => VisitStat.FindByLevelAndTimeAndPage((Int32)x.Level, x.Time, x.Page, useCache),
        e => e.Page = m.Page);

    if (entity == null) continue;

    entity.Times++;
    entity.Cost += m.Cost;
    entity.Save();
}
```

## 7）常见问题

- **统计重复**：时间未规整或唯一索引缺失。
- **并发异常频繁**：唯一键过粗，建议增加维度字段。
- **层级错乱**：`Fill(ps)` 后未再调用/保留规整时间。

## 8）落地建议

- 先做 Day 级，验证后再扩到 Month/Year。
- 写入链路尽量短，复杂计算放离线任务。
- 指标字段优先使用累加方式，减少跨表查询。
