# 树形实体（EntityTree）

树形结构是业务系统中最常见的数据形式，如权限菜单、行政区划、商品分类、组织架构等。XCode 通过 `EntityTree<TEntity>` 基类，让实体类天然具备树形遍历与操作能力，无需手工递归查询。



## 快速开始

在 `Model.xml` 中设计树形表，通常包含主键、父键、名称、排序四个字段：

```xml
<Table Name="Area" Description="行政区划">
  <Columns>
    <Column Name="ID"       DataType="Int32"  Identity="True" PrimaryKey="True" />
    <Column Name="ParentID" DataType="Int32"  Description="父级" />
    <Column Name="Name"     DataType="String" Description="名称" />
    <Column Name="Sort"     DataType="Int32"  Description="排序" />
  </Columns>
</Table>
```

生成的实体类继承 `EntityTree<Area>`：

```csharp
public partial class Area : EntityTree<Area>
{
}
```

> 对整型主键的表，继承 `EntityTree<TEntity>` 即可（等价于 `EntityTree<Int32, TEntity>`）。
> 非整型主键（如 `String`）使用 `EntityTree<TKey, TEntity>`。



## 关键配置（Setting）

框架通过静态属性 `Setting`（实现 `IEntityTreeSetting`）自动推断树结构，以下字段按约定自动匹配：

| 配置项 | 含义 | 默认推断规则 |
|---|---|---|
| `Key` | 主键字段名 | 实体唯一主键 |
| `Parent` | 父键字段名 | `"Parent" + Key`，如 `ParentID` |
| `Sort` | 排序字段名 | `Sorting`、`Sort`、`Rank`、`DisplayOrder`、`Order` 之一 |
| `Name` | 名称字段名 | `Name` 或第一个非自增字段 |
| `BigSort` | 大数排前 | `false`（默认小数排前） |
| `MaxDeepth` | 最大深度限制 | `0`（不限制） |

需要覆盖默认推断时，在实体静态构造函数中修改：

```csharp
static Area()
{
    Setting.Sort = "DisplayOrder";
    Setting.BigSort = true;
}
```



## 核心属性

| 属性 | 说明 |
|---|---|
| `Parent` | 父节点实体（从缓存获取） |
| `Childs` | 直接子节点列表（从缓存获取，带排序） |
| `AllChilds` | 全部子孙节点列表（深度层次树结构输出） |
| `MyAllChilds` | 全部子孙节点列表，包含当前节点自身 |
| `AllParents` | 全部父节点列表（从当前向根方向，不含当前） |
| `Deepth` | 节点深度，根节点为 0，第一级为 1 |
| `NodeName` | 节点名称（取 Setting.Name 字段值） |
| `ParentNodeName` | 父节点名称 |
| `TreeNodeName` | 带全角空格缩进前缀的节点名（用于下拉列表展示） |
| `TreeNodeName2` | 带 `|- ` 符号缩进的节点名 |
| `FullPath` | 反斜杠分隔的完整路径（包含自身），如 `\中国\广东\广州` |
| `FullParentPath` | 反斜杠分隔的父路径（不含自身） |

> `Childs`、`AllChilds` 等属性均通过实体列表缓存获取，不会产生额外数据库查询。



## 静态方法

```csharp
// 虚拟根节点（主键为空值），Root.Childs 即顶级节点列表
var roots = Area.Root.Childs;

// 获取整棵树（深度优先排列）
var tree = Area.Root.AllChilds;

// 查找某节点的直接子节点
var children = Area.FindAllByParent(parentId);

// 查找某节点及其所有子孙（以深度层次树结构输出，含根节点）
var subtree = Area.FindAllChildsByParent(parentId);

// 查找没有父节点的孤立节点（数据错误）
var orphans = Area.FindAllNoParent();

// 获取整棵树，排除指定分支（常用于选择父节点时过滤自身及子树）
var selectable = node.FindAllChildsExcept(node);
```



## 路径查询

```csharp
// 按路径查找节点，路径格式：中国/广东/广州
var node = Area.FindByPath("中国/广东/广州");

// 获取完整路径字符串（默认分隔符 /）
var path = node.GetFullPath();        // 中国/广东/广州
var path2 = node.GetFullPath(false);  // 中国/广东（不含自身）
```



## 典型使用场景

### 构建下拉选择树

```csharp
// 返回带缩进的节点列表，适合绑定到下拉框
var list = Area.Root.AllChilds
    .Select(e => new { e.ID, Name = e.TreeNodeName })
    .ToList();
```

### 获取面包屑路径

```csharp
var area = Area.FindByID(510100);  // 成都市
var breadcrumb = area.AllParents
    .Reverse()
    .Select(e => e.NodeName)
    .Concat([area.NodeName]);
// 中国 > 四川省 > 成都市
```

### 删除节点时检查子节点

```csharp
if (node.Childs.Count > 0)
    throw new InvalidOperationException("请先删除子节点");
```



## 注意事项

- 树形实体强依赖**实体列表缓存**（`Session.Cache`），整张表会加载到内存，适合节点数量不超过数万的小型树。超大树（百万节点以上）建议按需分段查询，不直接使用树形基类。
- `Root` 是一个虚拟空节点，主键为空值，不对应数据库记录。
- 数据变更后缓存自动失效（通过 `OnDataChange` 事件驱动），无需手动刷新。
