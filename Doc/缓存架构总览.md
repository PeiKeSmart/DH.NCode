# 缓存架构总览（Cache）

XCode 缓存体系是“按场景分层”的，不是单一缓存：

- `Meta.Count`：表行数缓存
- `EntityCache`：整表实体缓存
- `SingleEntityCache`：单对象字典缓存
- `FieldCache`：字段聚合缓存
- `DbCache`：数据库键值缓存
- `DataCache`：运行时缓存落盘（冷启动加速）

## 1）如何选型

### 实体缓存（整表）

- 场景：小表、字典表、读多写少
- API：`Meta.Cache.Entities`
- 文档：`实体缓存EntityCache.md`

### 单对象缓存（按主键）

- 场景：按ID/唯一键高频查询
- API：`Meta.SingleCache[key]`
- 文档：`单对象缓存SingleCache.md`

### 字段缓存（聚合）

- 场景：分类 TopN、下拉统计
- API：`new FieldCache<TEntity>("Field").FindAllName()`
- 文档：`字段缓存FieldCache.md`

### 数据层缓存（数据库实现）

- 场景：跨进程共享、无外部缓存组件
- API：`DbCache.Set/Get`
- 文档：`数据层缓存DbCache.md`

## 2）一致性策略简述

- 写操作会触发缓存失效或标记过期
- 多数缓存采用“先返回旧值 + 异步刷新”策略保障吞吐
- 强一致需求可在关键路径使用强制清理

## 3）性能与风险平衡

- 命中率越高，数据库压力越低
- 缓存粒度越细，一致性成本越高
- 错误选型（大表用整表缓存）会导致内存和刷新抖动

## 4）推荐实践

1. 先从单对象缓存切入（收益高、风险低）。
2. 字典小表再启用实体缓存。
3. 统计页面使用字段缓存，不要每次实时 group by。
4. 需要共享时再引入 DbCache，避免过度依赖数据库做热点缓存。
